# Frontend Implementation Plan: Obligations & Investments

This plan describes the UI/UX requirements for the new Obligations and Investments modules in the Frofinx Finance Tracker, based on the backend API changes in `frofinxAPI`.

---

## 1. Obligations Module (Debts & Loans)
**Endpoint:** `/obligation`
**Core Concept:** A unified interface for managing money you owe (Debts) and money owed to you (Lendings), including installment plans.

### Views to Create:
1.  **Obligations Dashboard/List:**
    - Filterable list by `direction` ('debt' vs 'lending'), `status` ('active', 'settled', etc.), and `isInstallment`.
    - Cards showing: Name, Counterparty, Principal vs. Remaining Balance, Progress bar (Paid vs. Total), and Due Date.
    - Aggregated "Summary" header: Total Debt, Total to Collect, and Net Position (using `GET /obligation/summary`).

2.  **Creation Form:**
    - Toggle for "I Owe" (Debt) vs. "Owed to Me" (Lending).
    - Fields: Name (e.g., "Car Loan"), Counterparty (e.g., "BPI Bank"), Principal Amount, Start Date.
    - **Installment Sub-form (Conditional):** If "Is Installment" is checked, show fields for:
        - Total Installments (e.g., 12)
        - Monthly/Weekly/Yearly selector
        - Installment Amount (auto-calculated hint: Principal / Count)
    - Optional: Wallet selector (if selected, backend auto-creates the initial money movement).

3.  **Detail View:**
    - Transaction history for this obligation.
    - List of linked Bills (if it's an installment plan).
    - "Record Payment" Button (triggers `POST /obligation/record-payment`).

---

## 2. Investments Module
**Endpoint:** `/investment`
**Core Concept:** Tracking portfolio value over time and recording returns/dividends.

### Views to Create:
1.  **Portfolio Overview:**
    - Summary (using `GET /investment/summary`): Total Invested, Total Current Value, Total Gain/Loss, and ROI %.
    - Breakdown by Type chart (Stocks, Crypto, Real Estate, etc.).
    - List of active investments.

2.  **Investment Creation Form:**
    - Type selector: Stocks, Bonds, Crypto, Real Estate, etc.
    - Principal Amount, Start Date, Platform (e.g., "Binance", "Col Financial").
    - Wallet selector (funds the investment).

3.  **Detail & Value Tracking:**
    - **Value Growth Chart:** Line chart using data from the `valueHistory` array.
    - "Update Current Value" Button: Modal to input a new price/valuation (`POST /investment/update-value`).
    - "Record Return" Button: For dividends or interest payments (`POST /investment/record-return`).
    - "Sell/Close" Button: To liquidate and return funds to a wallet (`POST /investment/sell`).

---

## 3. Integration with Existing Components
- **Bill Calendar:** Installment bills from obligations will now automatically appear in the existing calendar view since they are stored in the same `Bills` collection.
- **Wallet Balances:** Any payment recorded via the new modules will automatically update the selected Wallet's balance.
- **Transaction List:** Initial funding and subsequent repayments will appear in the main transaction stream with descriptive labels auto-generated by the backend.

---

## 4. API Reference for Frontend
- `POST /obligation/create`: `{ direction, name, counterparty, principalAmount, startDate, isInstallment... }`
- `POST /obligation/record-payment`: `{ id, amount, walletId, date }`
- `GET /obligation/list`: `?direction=debt&status=active`
- `POST /investment/update-value`: `{ id, value, notes }`
- `POST /investment/sell`: `{ id, saleAmount, walletId }`
